FDD extraction operating rules

- One PDF per run.
- Work on the current PDF only unless the task explicitly says to compare with a prior run.
- Preserve uncertainty, contradictions, and unresolveds.
- Do not invent facts, numbers, clauses, page ranges, or exhibit contents.
- Tables, notes under tables, exhibits, addenda, financial statements, guaranties, and franchisee lists are mandatory evidence objects.
- No canonical field without supporting evidence.
- If something exists but is not directly surfaced, say so explicitly.
- Prefer writing output files over long inline responses.
- Prefer targeted retries over broad re-reading.
- Do not run broad retries unless an audit, publish gate, or regression check identifies a material gap.
- Use specialist subagents only for narrow recovery work, regression recovery, conflict adjudication, or hard-family decoding.
- Do not use specialist subagents as the default first reader.
